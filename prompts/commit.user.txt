아래 변경 내용을 분석해서 Git 커밋 메시지(제목 + 본문)를 작성해라.

우선순위:
1) CHANGE OVERVIEW(통계/상태)로 변경 범위를 먼저 파악한다
2) DIFF와 신규 파일(untracked) 내용을 종합해 가장 중심 변경 1가지를 고른다
3) 제목에는 그 “중심 변경”을 반영한다 (특정 파일명만 나열하지 말 것)

type 선택 가이드:
- fix: 버그/오작동/에러 수정
- feat: 사용자 기능 추가
- refactor: 기능 동일, 구조 개선
- chore: 빌드/스크립트/도구/의존성/설정/인프라 작업
- docs/test/perf: 해당 성격일 때만

추가 규칙:
- 추측 금지: 입력에 없는 내용은 절대 쓰지 말 것
- 결과는 반드시 한국어로 작성할 것
- 코드 블록(``` )으로 감싸지 말 것
- 본문은 최대 3~6개 bullet로 요약할 것

출력 형식(반드시 준수):
<type>: <한글 요약>

- ...
- ...

입력:
{{INPUT}}
